NAME

initlocation - 创建一个从属的 PostgreSQL数据库存储区

SYNOPSIS

initlocation directory

DESCRIPTION 描述

initlocation 创建一个新的PostgreSQL从属数据库存储区。参阅 CREATE DATABASE [create_database(7)] 里关于如何管理和使用从属存储区的讨论。 如果参数不包含一个斜杠而且也不是一个有效的路径， 它会被认为是一个环境变量而引用。参考下面的例子。


 为使用这条命令，你必须以数据库超级用户登录（例如，使用 su）

EXAMPLES 例子


 用环境变量在一个可选位置创建一个数据库： 

$ export PGDATA2=/opt/postgres/data

 启动并停止postmaster这样它就能看到PGDATA2环境变量。 系统必须配置为postmaster每次启动都能看到PGDATA2。 最后∶ 

$ initlocation PGDATA2
$ createdb -D PGDATA2 testdb


 或者，如果允许你使用绝对路径，你可以这样： 

$ initlocation /opt/postgres/data
$ createdb -D /opt/postgres/data/testdb testdb
