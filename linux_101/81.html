传统拨接还是 ADSL 宽带？

当前台湾地区的网络挂机方式中，除了学术网络可直接连上因特网之外，一般的用户或企业用户则必须通过电话拨接，或者是近来流行的宽带系统（或者说成是拉专线）进行挂机。由于传统的电话拨接系统之传输速率实在太慢了（最高到 56 Kbps ），所以才会有 ADSL 或者是 Cable Modem 等宽带系统的生成。简单的讲，所谓的『宽带』其实是利用较新的技X，以让用户在拨接时获得比较高流量的数据传输。
那连上因特网之好处为何？为何大家一窝蜂 的想连上因特网呢？由于因特网上的丰富信息，你可以想象不用出门便知天下事的情况，例如，研究生查询paper数据再也不用直接跑到图书馆去，也可以 使用关键字查询的方法来寻找个人适合的信息。因此，因特网可以预见的，会越来越热门，尤其是下一波传统商业行为配合因特网销售行为的消费型态！更有甚 者，如果你想要自己架设一些站点的话（例如WWW站、FTP站、mail-server站等等），那就一定要连上因特网啦！
传输速率的问题：
这里就生成了一个问题，你连上网络后，实际的数据传输行为或许你可以不理（就是一些IP协议等等的），但是实际有关的下装速度你可能就不会不 care 了吧！传统的电话拨接网络，最高仅可至 56Kbps（这个 Kbps 是 Kbit/second ，当实际转换成 KBytes 时，只有每秒  7KBytes 的传输量！）如果你要下装的数据超过 10MBytes 时（例如某些驱动程序，即使压缩档也会到10MB以上！），换算一下，你可能需要 25分钟 以上的时间才能下装完毕！所以才会有越来越多的网络族群申请数据传输速度较快的 ADSL 或cable等宽带传输的服务！
另外有一个最大的问题，当前在网络上所谓的传输速率均是使用 Kbps ，亦即是 Kbits / second，与我们一般在 PC 上头文件总管上看到的文件大小（KBytes）是不一样的，因为 8bits=1byte ，所以你所看到的网络传输速率都要在除上 8 这个因子。举个例子，当前市面上看到的网络卡所标示的网络速率是 10/100Mbps，这个 bps 就是 bits/second ，因此，实际上的传输速率是 1.25/12.5 MBytes/second；而一些固网公司提交的计时制 ADSL 之传输速率可到 512/64 Kbps 也同样要在除上 8 ，就变成了 64/8 KBytes/second 了！
传统拨接还是 ADSL 拨接系统？
很多人都有疑问在这里！『我是否需要 ADSL 这套设备？』基本上，如果你是常常上网的族群，而且你又很不喜欢等待，那最好申请一下宽带系统吧，因为传统拨接系统是一定无法满足您的！但如果你只是偶尔才上一下网络的话（可能一个月不到 10 小时），那根本不需要 ADSL 系统，因为你用不着那样的速度呀！而且 ADSL 系统就算你都不使用，每个月还是得负担一定的费用！当然，如果你有钱的话就没啥大不了的罗！

ADSL 的两三事

下面是关于计时制 ADSL 的相关说明（数据来源在中华电信的网页）：
ADSL 的全名为"Asymmetric Digital Subscriber Line"，它是一种利用传统电话线(亦称为双扭线)来提供高速因特网上网服务的调变/解调变技X。在提供上网服务的同时，并可提供一般电话语音服务。 ADSL的关键观念在于其上行与下行的带宽是不对称的，事实上，从网络提供者到用户家(俗称下行)的带宽是比较高的，这样的设计匹配了一般使用因特网的 使用习惯与特性，也就是用户接收的数据量远比其送出的数据量来得多。
而计时型ADSL则是利用PPP over Ethernet 通信协议，在ADSL网络配置上认证用户、记录用户的挂机时间，并取得动态IP。对一般家庭及个人用户提供更经济价格的上网服务。
由上面的说明可以看出，ADSL 具有比较快的因特网数据传输速率，而计时制的 ADSL 拨接系统是在用户拨接上网时才予以配发 IP （可以想成是你的计算机在网络上的门牌），这个 IP 是随机给的，亦即你每上次网可能都会使用不同的 IP ，这样一来有个好处，由于用户不太可能 24 小时均在联机，因而或许可以 100 个人使用 50 个 IP ，如此将可增加 ISP 的 IP 调度！但是这也造成一个问题，就是当你需要架站的时候，由于你的 IP 不固定，将导致别人无法连接上你的网络系统（当然，这可以经由动态 IP 之 DNS 系统来克服）。不过，如果你并不想要架站，只想上网打打 Game 或者浏览一下网页，那，计时制的 ADSL 是比较适合您的！当然，架站比较好的选择仍然是固定制的（不过，如果您跟 VBird 一样是穷学生的话，申请计时制的 ADSL 仍然是可以架站的喔！）。
动态 IP 、固定 IP 、物理 IP 与虚拟 IP？
讲了老半天，计时制与固定制不就是 IP 取得的方式不同吗？那 IP 是啥？有啥 IP 需要注意的呢？以下简单的说明一下。
  • 物理  IP：在网络的世界里，为了要识别每一部计算机的位置，因此有了计算机 IP 地址的定义。一个 IP 就好似一个门牌！例如，你要去微软的站点的话，就要去『 207.46.197.101 』这个 IP 位置！这些可以直接在因特网上沟通的 IP 就被称为『物理 IP 』了。
  • 虚拟  IP：不过，众所皆知的，IP 地址仅为 xxx.xxx.xxx.xxx 的数据型态，其中， xxx 为 1-255 间的整数，由于近来计算机的成长速度太快，物理的 IP 已经有点不足了，好在早在规划 IP 时就已经预留了三个网段的 IP 做为内部网域的虚拟 IP 之用。这三个预留的 IP 分别为：
      A级：10.0.0.0 - 10.255.255.255
      B      级：172.16.0.0 - 172.31.255.255
      C      级：192.168.0.0 - 192.168.255.255
    上述中最常用的是192.168.0.0这一组。不过，由于是虚拟 IP ，所以当您使用这些地址的时候o当然是有所限制的，限制如下：
    • 私有地址的路由信息不能对外散播
    • 使用私有地址作为来源或目的地址的封包o不能通过Internet来转送
    • 关于私有地址的参考纪录(如DNS)o只能限于内部网络使用
    由于虚拟 IP 的计算机并不能直接连上 Internet ，因此需要特别的功能才能上网。不过，这给我们架设IP网络做成很大的方便o比如s即使您当前的公司还没有连上Interneto但不保证将来不会啊。如 果使用公共IP的话o如果没经过注册o等到以后真正要连上网络的时候o就很可能和别人冲突了。也正如前面所分析的o到时候再重新规划IP的话o将是件非常 头痛的问题。这时候o我们可以先利用私有地址来架设网络o等到真要连上intetnet的时候o我们可以使用IP转换协议o如 NAT (Network Addresss Translation)等技Xo配合新注册的IP就可以了。
  • 固定 IP 与 动态 IP：基本上，这两个东西是由于近来网络公司大量的成长下的产物，例如，你如果向中华电信申请一个商业型态的 ADSL 专线，那他会给你一个固定的物理 IP ，这个物理 IP 就被称为『固定 IP 』了。而若你是申请计时制的 ADSL ，那由于你的 IP 可能是由数十人共同使用，因此你每次重新引导系统上网时，你这部计算机的 IP 都不会是固定的！于是就被称为『动态 IP』或者是『浮动式IP』。基本上，这两个都是『物理IP』，只是网络公司用来分配给用户的方法不同而生成不同的名称而已！

计时制还是固定制？

如果您已经确定要使用 ADSL 来作为连接 Internet 的系统，那下一个问题是：要使用较便宜的 计时制 还是较贵但是有固定 IP 的 固定制 呢？若以省钱为导向，那不用说，一定是申请计时制的 ADSL 系统，因为确实有比较便宜；若以目标为导向，如果你需要稳定的架站环境，自然选择固定制的会比较好；那若以网络带宽分享为导向呢？例如你在宿舍大约有 5 个人要使用一条专线，或者是家中有少于 5 部计算机使用一条电话线挂机至 Internet ，那最好是选择计时制的 ADSL ，因为计时制的 ADSL 在中华电信可同时开放两个 IP 而 Seednet 更可开放到 5 个 IP 让同一个帐号上网使用，如此，则挂机分享仅需要一棵不用 1000 元的集线器 (Hub) 就可以罗！如使用固定制的话，由于只有一个固定的 IP ，则你要使用挂机分享带宽，就必须要比较理解挂机的方式，或者加装 IP 分享器，或者自己架个具有 NAT 功能的小站了！
如果你肯定要使用 计时制 的 ADSL ，又想要使用带宽分享，以下是需要注意的几个地方：

    • 少于 5 部计算机要挂机：如果只有少于 5 部计算机要使用这条 ADSL 线，那直接申请 Seednet 或 Hinet （中华电信仅提供两个 IP 喔），再加装一个 Hub 作为 ATU-R 及 PCs 的连接中介就可以分享了；

    • 多于 5 部计算机要挂机：多于 5 部计算机想要同时挂机的话，大概只有两个方法：

      • 加装一个 IP 分享器以及多孔的 Hub ：利用 IP 分享器来分享同一个 IP 的网络系统，不过，通常 IP 分享器不是很便宜！

      • 自己架一个 Server：这是比较合适的分享方法，而架 Server 的方式有很多种，简单可分为两种：（1）直接以使用中的计算机加装一块网络卡（亦即这部计算机中有两块网络卡）来作为 Server ，不过这个功能当前仅能在 Windows 2000 这个系统中使用（好象 windows me 也可以，但鸟哥没试过）；（2）以淘汰掉的旧计算机来架个 Linux 站，并打开 NAT 功能，即可达到带宽分享的目的：这是当前鸟哥家中使用的带宽分享方法，不但计算机一引导系统立即上线（不用再拨接），而且也没有同时一堆人挂机就有点延迟的现象！
