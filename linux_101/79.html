关于本网页最初的目的

如果大家之前已经常常到『鸟哥的 Linux 与 ADSL 文档』当中参观的话，那幺应该对小弟（ VBird 啦 ）不陌生才对。由于小老弟我一向记忆力就不好，所以当初才会写了那样的一个站点！最主要的目的是因为要提供 VBird 自己一个可以随时在联机参考自己以前作法的一个数据来源。由于 VBird 喜欢玩计算机、网络等等的咚咚，所以有些朋友常会请求 VBird 帮他们建立一下站点，而因为如同前面说的， VBird 的记性一向不好，忘性又特别佳，为了害怕自己在帮别人安装站点的时候，突然忘了应该要怎样作（那不就糗了！），呵呵！那幺当初写的一些文档不就很有帮助罗！对啦！当初的目的就是要提供 VBird 自己的翻阅之用！没想到一推出之后，还蛮受大家欢迎的，所以也就很高兴的继续小老弟的写作之路了！
此外呢，由于当前网络上等价流行『 ADSL 』这一种可以无联机网的拨接或者是固定制的挂机方式，这种系统最常也是比较被接受的是『 计时制 』的 ADSL 拨接的挂机方式，主要是因为他便宜了一些，并且最起代码的挂机速度可以到达 512/64 Kbps ，这里需要注意一下，网络上的传输速度常常以 bits/second （所以写成小写的 bps ）的方式来计量，这与我们常常说的文件大小的 Bytes 不相同，基本上你应该要知道的是 1Bytes=8bits ，所以将传输速度换算一下，拨接制 ADSL 可等价于 64/8 KBps （所以写成大写的 Bps）的流量，如果网络速度正常的话，那幺你的 ADSL 传输速度在下装方面应该会有 40-50KB 每秒的传输，这样的传输速度对于一般人来说已经是等价足够了！而由于这样大的传输速度，也就造成等价多的朋友们在想，咦！那我不就可以使用我的 ADSL 来架设站点了！？没错！正是如此，所以， VBird 才会渐渐的将自己的一些安装站点的过程写出来，提供大家一个安装站点的经验！
不过呢，由于当初写那些文档的时候，小老弟仅是一个 Linux 的新鲜人，狂妄的写了一些心得分享的东西，可能还会误导有心想学 Linux 的新人呢！伤脑筋！而且，当初写的东西主要是建立在 Red Hat 6.x 版本上的系统，而该系统在安装与架设上真的是很简单，很方便，也很容易上手！然而重大的问题出现了！就在我很高兴的玩着网络的时候，我们的主机竟然在不为任何人所知的情况下被怪客（Craker）入侵了，而帮朋友建立的站点竟然连 root 的口令都被更动过...... 甚至我们的上层主管机关还曾经来信问到：为何我们的主机会去攻击别人的计算机？？这个时候才知道，唉呀！功力不足呀！而且『安全观念』不正确！很多的安全机制都没有装上去，所以当初安装的主机一上线，几乎就等于一只美味可口的羊被丢到狮子群中去一般，当真是危险的很！
就如同我之前说的，由于 VBird 在安全的防护观念上不足，所以造成先前一些安全上的困扰，这也是当前大家常会遭遇到的问题！然而很多的朋友们会问到说：『拜托！我的网络不过是 512/64 Kbps ，谁会有兴趣来 入侵 我呀！？』是的，很多朋友们都是这么想的，这样想是没有错啦。不过，你知道怪客（craker）入侵你的计算机在作什幺吗？他们绝对不是对你的计算机主机有兴趣，入侵你计算机主机，最主要的目的可能有：
(1)当作训练自己电脑贼的功力，让别人知道自己的存在；
(2)没事，好玩，在网络上捉到的工具，就是试试看罗；
(3)以你的计算机主机当作跳板，去攻击怪客想要攻击的目标。
现在你知道问题的严重性了吗？还不知道？让 VBird 来告诉你吧：当怪客入侵了你的计算机，然后以你的计算机四处去攻击别人时，他所要下达的指令并不会耗去太多的网络资源，但是被怪客攻击的主机上面，所留下来的入侵记录却是『你的计算机 IP 』！理解了吗？当被入侵的主机提交告诉的时候，警调单位提调的可是『您』，而不会是怪客喔！那幺您仍然觉得你的计算机主机没有防护一点关系也没有吗？？
在经过了一年多来的摸索，虽然 VBird 还是一个 Linux 新鲜人，仍然无法与网络上的前辈们相比拟，不过，我仍然希望可以藉由自己的一些错误或者有用的经验，提供一下 Linux 的其他新鲜人们，一个可以学习的机会！当前， VBird 作这个网页的主要目的：
  1. 以较安全的方式安装 Linux ；
  2. 以较简易的方式提供新鲜人们安装的技巧；
  3. 提供一些经验给安装某些套件时的问题解决；
  4. 提供一些网络安全管理的小方法。

关于一些网络的资源

在这里我提供的只是一些简单的架设技巧，与一些相关的注意事项，并没有着墨在很多的理论上，而且， VBird 本身并不是信息专科的人才，基本上，对于理论方面的能力也是力有不殆啦！所以也就没有办法提供这方面的相关信息！并且，我有兴趣的地方在于提供 ADSL + Linux 的架设方面，所以着重的方式就在于此，如果您对于小弟所写的这个部分兴趣不高，并且希望理解更多的理论信息，那幺欢迎直接参考 Study-Area 的 网中人兄 写的 Linux 文章，相信收获将会更高喔！

关于本网页的参考方式

网页设计之初， VBird 想先绍介一下 ADSL 的挂机分享带宽的方式，然后慢慢的将 Linux 架站的观念导进来，我会建议：
  1. 先做成局域网：至少可以让你的局域网内的『网络上的芳邻』可以互相分享资源』；
  2. 决定带宽分享方式：并不是所有的人都需要架站的，如果您只是很单纯的希望可以上网查查数据而已，那幺实在没有必要花太多的时间在研究架站的问题上，买个 IP 分享器已经可以等价的满足您的需求了！但是，万一您希望可以在收发大量的信件上有一个较快速的网络，并且可以在任何时候都可以激活一些必要的服务，那幺实在没有其他的理由阻挡您架设一个完全可以让您一手掌控的主机！
  3. 决定主机服务的内容：很多的朋友架设完 Linux 主机之后，第一个安装的大概就是 Apache 吧，毕竟主机架设完成之后，总是要给人看呀！再来呢？则是希望可以开放 FTP ，VBird 一直觉得 FTP 实在是没有必要对外开放的，因为，你知道吗？你的 ADSL 上传带宽只有区区的 64 Kbps ，而且数据在上传的时候，将会影响下装的速度！如果被入侵了，更是伤害您主机呀！这个问题实在需要好好的思考一下，你需要的主机到底是哪一种型态呢？
  4. 做好安全分析：网络的维护是等价繁琐的，你得要常常上 Red Hat 的站点看看有没有新的信息，有没有安全的漏洞等等的问题，烦都烦死了！不过，这却是必要的，所以，你需要常常的分析您网络上的一些相关的讯息才行！
  5. 架设美美的网页：说穿了，架站最大的目的还是在吸引人群啦！所以这里 VBird 也稍微提供一些站点架设的套件安装！
大概就是这样了！当前我们提供的操作系统主要以 Red Hat 7.2 这一个最新的套件为主，开始来玩吧！

VBird 的成长过程：
小的时候家里住在海边的渔港，算是很传统的渔村小镇了，我们的渔村名字叫做红毛港，不是北部的红毛港，是在高雄旗津对岸的一个小 镇，这个小镇当前还是很传统的，为什幺会很传统呢？这是由于当年台湾还在经济起飞的时候，所以政府为了扩展高雄港口的腹地，因此就大张旗鼓的说是要『迁 村』要整个村子一起向外迁移，所以不准我们继续建设，因此，我们的小镇就一直保持当时的风貌，说的好听一点，是『保存传统的渔村风貌』，说的难听一点，就 是我们『大有为的政府』让我们红毛港的发展整整迟钝了30-40年！不过，也好在当时年纪小不懂事，所以住在渔村的日子过的倒是很开心的！我们当时是很贫 穷的一群，父母亲为了赚钱，没有空管我们，所以我们几个同年龄的小孩从小就好象野孩子，不必辛苦的念书，也从来没有补习，父母亲从来就不会逼我们念书，因 为他们也认为要读书还不如赶快国中毕业当兵去，当兵回来赶快去工作，好贴补家用！所以罗，小时候只要不惹老师生气就好了（因为我们小时候老师是很有『权 威』的，如果在学校被老师打，回家之后父母亲一定会再修理你一次！因为『惹先生生气一定是你不对！』然后隔天还会拧着你的耳朵到学校去请老师『息怒』，并 请老师『多加管教我们家小孩！』小时候就是这样，所以只要不惹老师生气，那我们要作什幺，大人们从来就不会管的！），也就因为如此，从小就很会制作一些不 用钱的童玩，筷子枪、报纸黏成的风~、自己制作的小钓鱼竿，然后堆土窑（就是很出名的『土窑鸡』那一种土窑，我很厉害喔！从小就很会生火！露营的时候大家 最喜欢跟我同一小队，因为我们的晚餐一定最快完成！）、打陀螺、玩『ㄤㄚ标』、、、、、反正小的时候虽然没钱，虽然过年没有很好吃的东西，没有很棒的新 衣，没有很多的压岁钱，但是有爸爸妈妈的爱心，有童年的一些快乐的回忆！唉呦！就是过得很无忧无虑就是了！不象现在的小孩这么辛苦.....
VBird 的求学过程
快快乐乐的过完一个无忧无虑的童年，接下来就是爸爸妈妈的伤脑筋时刻了！刚刚说了，爸妈本来的规划流程是这样：

没想到爸妈想的如意算盘到现在却还没有一个实现。首先，小时候笨笨的我，却没想到能够考上当时高雄第一自愿，然后还不小心考上国 立大学，本来想说，恩，读完大学总可以出去赚钱了吧，没想到还能考上研究所，恩！再不然，读完研究所可以回来工作了吧！嘿！现在还在念博士班！伤脑筋！结 果从小到大，还没赚钱给他们俩老人家花用过....真是不孝！不但如此，当前还没有结婚！当然也就没有小孩，还亏我是家里的老大，更伤脑筋了！除此之外， 家里还有三个小弟，这三个小弟在过年的时候最开心了，因为四个孩子刚好一桌，从来就不会缺牌友....又岔题了！他们常常会吓我，说『老哥，你要知道，现 在去当兵的都是 20 岁左右的年轻小伙子，你以后是当排长，想一想，你这个 30 多岁的老骨头要跟 20 岁的小毛头赛跑，就很替你担心』我咧......

到了大学，开始去搞社团，当时参加的是成大的『慈幼社』，到当前为止，我生活照最多的时期就是当社团干部（大二大三）那两年了！你一定会说！嘿！VBird 真是有爱心，跑慈幼社去照顾失亲或者是智能还待开发的小孩，其实，最早之前是因为听学长说那儿的漂亮女孩比较多....没想到，由于一群男人都是这么想，所以，我们那几届主要的成员都是男生......不过，反正 VBird  就是那种到哪里都可以生存的人，跑社团的次数多了也就开始乐在其中，其间还主办过一个营队的活动，反正就是四年没有白过就是了！到了大四，就开始念书，准备研究所，之后，就到现在了！咦！似乎有件事情没交代，没错，不要怀疑， VBird 当年是很纯情的，大学 + 研究所六年，没有交过女朋友......因为学弟跟我说：『学长，你是那种有女孩子缘，没有女朋友缘的类型』我咧....什幺理论！

接触到 Linux 的时候，算是我们 study area 里面最慢的了！但是也是最有压力的！因为我要靠他过日子....如何说呢？听我娓娓道来。因为我们研究室有兴趣要将美国环保署的一个空气质量模式，叫做 Models-3 的东西引进到台湾，但是实际上台湾并没有人会跑，所以我们得亲自到美国去找师傅....然后，最麻烦的是， Models-3 只能在 Sun 的 Unix 上面跑，我咧，吃到这么大，从来没有摸过 Unix 的机器。为了这一个计画案，一部近百万的 Unix 就这么买了下来，要注意喔，当时（2000/07）我对 Unix 根本就不知道什幺是什幺！更不要说是操作了，光想到一部近百万的机器在身边，里面的东西是什幺都不清楚，就觉得很惶恐！当时连 ls 是什幺都不知道....后来，好在大学的时候玩过DOS ，对于下指令的过程还不会很生疏，那幺，既然 Sun 的 Unix 不敢玩，那我就用旧计算机架一部 Linux 来试试看怎幺样！恩！不错的想法！就赶快付诸实行，安装好了 Linux 之后，操作了一些指令，觉得真不错用，那既然安装好了主机，学校的 E-mail 容量又这么小，那不然就自己作一个邮件主机好了，所以就用了 Red Hat 6.1 架了一个邮件主机，这是在去年 11 月份左右的事，然后，既然有主机了，我们研究室的毕业生也不少了，恩，为了大家联络方便，那就再架一个 Web 主机好了，那是在今年 2 月份左右的事，咦！既然研究室的备份数据这么多，那就再架一个 SAMBA 主机好了。之后，回到家中，几个小弟都想要安装 ADSL 来玩网络游戏，比较快嘛！恩！那就使用计时制的 ADSL 加上 NAT 主机来分享带宽好了，恩！既然 NAT 主机都架好了，为了国内外带宽不一样的情况，那就再开放一个 Proxy 服务来分流我们家里的网络流量好了！后来，在今年 7 月份左右，发现到我们的主机怪怪的，一查之下，竟然有电脑贼入侵过我们的主机，那时才又开始注意到网络安全的重要性，就开始摸索一些简单的安全防护观念。然后，因为做过的事情太多， VBird 的『记性』向来不好，『忘性』却又特佳，并且我总不能永远待在成大吧，研究室总是需要有人出来交接这些网络方面的问题，所以，就开始了『鸟哥的 Linux 与 ADSL 小站』的写作，写着写着，没想到就跟大家交流起来了，所以当前就是这样罗！唉呀！没有说到重点，差点忘了！由于玩 Linux 玩出了一点点的心得，加上自己曾经学过程序语言，因此对于逻辑概念是有一定的程度，所以，恭喜我吧！VBird 终于在今年完成 Models-3 的本土数据仿真的工作了！真高兴！不过，我还是比较喜欢 Linux ..... Sun 的机器实在太贵了，里面的 Unix 系统因为是老板的财产，也是我们主要的模式仿真环境，都不敢乱动....，不象 Linux 要怎幺玩都可以！
