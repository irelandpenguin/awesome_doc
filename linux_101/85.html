鸟哥的 Linux 与 ADSL 私房菜

挂机分享带宽方法的建议

最近更新日期：2002/01/27

看完了上面的几篇心得分享之后，相信大家对于以 ADSL 作为挂机分享的方法大致上应该有了一些理解了！好了，那幺现在来出个考题给大家作作..........喔！不是啦！是要大家来想一想，如果是『我自己要带宽分享』的话，应该要如何选择挂机的方法呢？以我而言，由于当前 ADSL 里面，最热门的应该还是 计时制 的拨接方法，因为他毕竟最便宜，一个月大约费用（2002/01/27日为止前）在 1100~ 1300 新台币之间，至于略高一级的固定制就需要这个价格的两倍！对于穷学生的我，以及上网的带宽不需要太高的人们来说，计时制的也实在是足够了！那幺如果要我建议『最简单』的带宽分享方式的话（大概就是针对计时制的罗），那幺我会建议如此的顺序：
  • 买 IP 分享器：
    其实 IP 分享器的功能就有点象是 Linux 的 NAT 主机，好一点的 IP 分享器不但提供直接以 Web 界面管理，以及同时具备 DHCP （自动分配 IP 地址的功能之服务的名称）的功能，另外，同时提供 Hub （或者是 Switch ）的连接孔，所以呢，一部 IP 分享器完全解决您上网的需求之需要！而且，要架设本地内的主机时，可以通过 IP 分享器的 port 对应设定来将某些服务封包转送到你的局域网内，也就是说，你使用了 IP 分享器之后，你的本地内的计算机『全部的 IP 都是 虚拟 IP』，但是你却可以『通过 IP 分享器的 port 对应』（简单的说，有点象是 Linux 的 NAT 设定罗）来让你的虚拟 IP 的计算机也可以架站，并且与 Internet 相连接！
    这是所有分享带宽的方法里面最简单的了，因为所有的拨接工作都交给 IP 分享器，你的计算机只要接到 IP 分享器上，就等于上网了，不过，就是费用上高了点，而且 port 对应的设定方法也比较麻烦一点。不过，如果你很有钱，而且也很怕麻烦，再加上你只是要分享带宽的话，那幺 IP 分享器是最简单的罗！
  • 申请 Seednet 或 hinet 等提供多个浮动式 IP 的 ISP：
    另外一个最简单的方式发生在『挂机的计算机数量不多』的情况下。据我所知，当前小于五部计算机的 ADSL 挂机分享方式里面，最简便的就是申请 Seednet 的计时制 ADSL 了！因为 Seednet 提供 5 个浮动式 IP 给一个帐号，所以，在你申请的线路上最多可以五部独立安装拨接程序来上网！只要买个 Hub 或者是 Switch 就可以了！如果没钱的话，买个不用 1000 元的 Hub 就可以分享罗！又简单又方便！所以，这里告诉大家，如果您的局域网内的计算机数量真的不多的时候，然后又没有所谓的架站问题，那幺使用 Seednet 可以说是最佳的解决方案了！当然罗，如果你的计算机只有两部的话，那幺 Hinet 也是蛮好的一个选择。至于使用 Giga 的朋友，虽然可以获得一个固定的 IP ，但是就无法使用此一方法罗！
  • 建立 Linux 或 Windows 2000 主机：
    如果，真的，万一，不幸，你选择了类似 giga 的拨接制，或者是你的计算机数量超过了 5 部以上，那幺要如何来分享你那小小的 ADSL 带宽呢？没办法了，除了 IP 分享器这个最简单的方法之外，就是架设 Windows 2000 或者是 Linux 这种主机来分享罗！使用Windows2000 提供分享较为简单，你只要有 Hub 配合每部计算机一个网络卡，就可以分享了，不过缺点是，你还需要在每部用户端计算机上设定 MTU 的值！比较麻烦一些些！
    而使用 linux 架设服务器是我最喜欢的方式了，不过，也是所有分享带宽的方法 里面，技X性较高的一种方法，不过，以 Linux 架设主机有等价多的附加价值，就是可以架设非常多的服务器，并且，个人认为，以 Linux 主机架设服务器的带宽分享速度是所有方法里面几乎是最快的（请注意，其实 IP 分享器也是主机的一种！只是他的功能被简化而已！），并且，你只要有不要用的 p-100 左右档次的计算机配合 500 MB 的硬盘与 32MB 的 RAM 就可以架设了！而主机也不需要屏幕、键盘、鼠标、甚至，安装完毕之后，连显示卡都可以拔掉不使用！只要有『主机板、CPU、RAM、硬盘、网络卡』就可以 了，当然，安装的时候这些东西都还是需要的，只是安装完毕之后，就可以拔掉了！并且，Linux 的资源耗费的非常的少，所以你也不用担心 CPU 会烧掉的问题！
    使用主机来架设带宽分享的方式可能是弹性最高的一种了，其中，Linux 分享的情况更是如此，因为你的配备不需要太高，并且，一块网络卡可以分享的计算机理论上是『无限多部』的，（当然罗，由于数据封包的碰撞问题，自然不会接太多部计算机在一个网络卡上面！）而且 Linux 也提供转 port 的功能，只要几个指令就可以达到这样的功能，好的很！并且，速度要比 Windows 2000 快的多罗！
  • 使用软件来仿真主机：
    这是我最不喜欢的挂机方式，因为问题多多，并且安装方面也需要蛮大的调试功夫！而挂机方式也是所有方法里面最慢的！容易 Delay ，此外，如果主机有点小问题（其实，哪一部 windows 系统的计算机会没有『小问题』的？）那幺你的用户端计算机，呵呵！跟着一起挂！超不稳定的！不过，这个方法是最便宜的方式了，因为只要 windows 98 的系统就都可以使用！
好了！说了这么多关于『简单的』挂机方式建议，那幺有没有『最推荐的方式』建议呢？当然有罗！最推荐的方式自然就是 Linux 的架站罗！理由有几点：
  • 以 Linux 进行拨接来分享带宽，他网络的速度可以说是最快的一种了，在我的经验里面，几乎不会有 delay 的问题，而且等价的稳定；
  • 以 Linux 进行分享的配备可以使用废弃不用的 486 以上档次计算机就够了，可以达到废物再利用的『环保概念』，呵呵！此外，硬件配备几乎并不会影响到网络传输速度的问题；
  • 架设 Linux 可以让你的一些计算机观念与网络基础更清楚，因为所有的设定都可以修改，以匹配你的需要，所以，当然可以更清楚的理解你的需要罗！
  • Linux 的附加价值等价的高，你只要将硬盘加大一些些，并将 RAM 也扩展一些些，就可以进行很多的主机架设，我最常架设的就是 mail server 了！假如你今天要将十份数据寄给同一个人，而且你的时间又不多，并且也很怕会影响到其他人使用网络的权力，那幺你只能在夜深人静的时候才能寄信了，而且一封 5MB 的信件可能要等待超过 10 分钟的时间才能传送完毕！要寄 10 封的时候，哈哈，想不疯掉也很难，真可怜！如果架设了邮件主机之后，那幺你只要按下传送键，由于对内的 Hub 最低也有 1.2MBytes/second 的速度，呵呵，机秒钟就传送完毕了，你就可以关掉你的 Windows 机器去睡觉罗，其他的邮件传送全部交给 Linux 主机去负责，呵呵，轻松又方便！此外，人家寄大量信件给你的时候，由于是直接寄到你的 Linux 主机上面，所以你在收信的时候根本就不需要几秒钟就可以收受完毕！并且不需要连接到 Internet 上面，呵呵！很简便吧！
  • Linux 可以设定检测的时段，使断线了之后可以立即拨接，这个功能在 Windows 底下也有对不对，但是呢，Linux 可以在 IP 被改变了之后，立即以 e-mail 通知你，让你可以随时知道你的 IP 情况，这个功能就不简单了吧！此外，在你于外部计算机中（例如在公司内部）也可以通过 Internet 挂机到你家中去进行修改的动作！呵呵！很方便吧！
基于这些理由，所以我最喜欢的方是就是 Linux 分享带宽了！但是，并不是所有人都适合架设 Linux 的，例如如果你是一家公司的主管，平时忙都忙毙了，还要架设 Linux 吗？可能没时间对不对！是的，架设 Linux 真的需要花一些时间，所以没有耐性的朋友们，呵呵，先想其他的方法吧！好了，那有没有第二推荐？！当然有罗，可以选择 Windows 2000 这个操作系统来分享呀！他的理由有几点：
  • Windows 2000 当前很多都是『随机版』，所以是正版的软件，这个对于一般的公司行号是很重要的！
  • Windows 2000 的使用方法与界面与 windows 98 或 windows xp 几乎完全一模一样，你根本就不需要在花时间去学新的东西，所以很适合立即的架设分享带宽！
  • Windows 2000 的架设方法很简单，接线方法就更简单了，另外，你可以使用你当前既有的设备来架设 Windows 2000 的挂机分享机制，不需要在花附加的设备（当然， Hub 是一定要的啦！），既省钱，又简单！何乐不为？
好了，那有没有适合懒人的方法。ㄟ！当然有，买 IP 分享器吧！除了贵了一点，个人觉得 IP 分享器算是一个不错的解决方案罗！
