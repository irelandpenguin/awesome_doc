目的与适用性

目的：
如果你不想使用服务器（Server）的配置来挂机分享带宽的话，可以使用一些挂机分享的软件来实现带宽分享的目的。使用挂机分享的软件时，挂机的方法与使用 Hub 同时连接所有的计算机是一样的，而且做为主机的机器可以不用插两块卡，仅只要一块网络卡就可以达到挂机分享的目的。
下装：
VBird 是在很久以前拿到这套试用软件的，VBird 也不知道这家公司还在不在....你可以在这个网页上下装，请按这里下装。
适用性：
基本上，本方法适用于只有一个固定 IP 的情况（例如 Giga ADSL），当然，也适用于其他的计时制拨接 ADSL 系统。
缺点：
虽然这个是最便宜的方式，而且也不需要更新您的系统，因为在 Windows 98 底下就可以进行这一种带宽分享罗！但是，由于这个方法是使用软件来控制硬件，所以当做为主机的 PC 有问题时，会造成等价大的困扰！并且，由于是软件来控制硬件，基本上，在传输速率方面，可能作为用户端的计算机会有较慢的情况发生！说实在的，除非您真的不愿意花时间在 Linux 或 Windows 2000 底下架设 NAT，又或者您不想要购买 IP 分享器，否则 VBird 真的不建议安装这一种带宽分享耶！
原理：
说穿了，这种方法的工作原理也不是很难，这类软件只是将你的网络卡另外再设一个子网段的虚拟 IP ，利用这个『虚拟IP』来分享你的网络带宽罢了！在 Linux 当中就有这一种方法的使用哩！

Windows 系统下安装网络卡

      其实，安装网络卡的方法都一样，就是：

        （1）将硬件安装好；
        （2）引导系统让 Windows 找驱动程序；
        （3）Windows 找不到驱动程序的话，就放入驱动程序片，让 Windows 去捉；
        （4）如果还不行安装的话，就直接到『开始』->『设定』->『控制台』->『新增移除硬件』去搜索一下罗！

      详细的安装方法可以到使用 Hub 挂机分享那一篇去看看。

增加通信协议（确定网络上的芳邻可正常工作）：

基本上，在你完成了硬件的挂机以及网络卡的安装之后，如果没有意外的话，通常 Windows 内的『网络上的芳邻』就可以工作了，亦即是你可以在你的计算机中看到你『局域网』内的所有计算机。如果不行的话，应该就是网络有点问题了。要确认问题的所在，你可以先：
  1. 确定通信协议都正常：
  2. 确认挂机设备完全没有问题：
总而言之，在安装这套挂机分享程序之前，你一定要：
  1. 先确认你的 『局域网』 没有问题，而且，
  2. 做为主机的那台计算机也可以正常的拨接上 Internet 才行！
否则如果出问题，你一定会不晓得是哪一个部分出现问题的！

 10. 安装完成之后，请重新引导系统吧！
 到此为止，主机端的设定就 OK 了！在你重新引导系统之后，你会发现屏幕右下方多了一个有点象是恐龙的小图示，那就是安装完成啦！这时候你要做的就是请拨接上网吧，拨接完成之后，请确定你的 Internet 是问题不大的，然后请再继续安装你的用户端 PC ！
注：最重要的是第三步骤，请必定选择 Base Computer！

做为用户端（Client）的 PC 安装 All Aboard

安装用户端的计算机就更简单了。由于主机在安装完了 All Aboard 之后，就成了具有 DHCP 服务的主机，所以在你的用户端计算机上，安装 All Aboard 就很简单。
 1. 在上面的第三个步骤中，如下图所示，由于我们是安装用户端计算机 (Client PC)，所以请选择『2nd Computer(Client Computer)』这一项。
 2. 然后与上面的步骤都一样，等到出现下面的屏面时，由于你的计算机可以直接由主机来供应你的 IP ，所以选择『Set the client to obtain an IP Address automatically』这一项即可。
然后又与安装主机时相同的讯息，（但是步骤被省略很多了），反正最后就是重新引导系统了！
3. 重新引导系统之后，你的这部用户端计算机应该就可以直接通过主机来挂机罗！如果不行的话，应该就是你的 IP 没有捉对，这时候在你的 Client 计算机上：
『开始』-> 『运行』-> 输入『winipcfg』
会出现一个小框框的屏面告诉你，这部计算机的 IP 情况。看看你的IP是否为 192.168.244.151，如果前面三位数字不对的话，请按
『全部释放』
『重新捉一次IP』（朝意思一样的钮就按下去吧），
那用户端应该就可以捉到你的主机了！
